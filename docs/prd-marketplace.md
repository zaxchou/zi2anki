# PRD: 市场功能 — zi2anki 产品重定位

## 产品目标

**一句话定位**：zi2anki 从个人 SRS 工具升级为面向书法学习者的字库记忆平台 — Admin 统一维护字帖库，用户免费订阅学习。

**三个核心目标**：
1. 用户无需自建牌组，直接从市场订阅上百个书法字帖
2. 卡片内容共享一份，学习进度各自独立，节省存储
3. Admin 集中管理字库（导入/编辑/发布），用户端零门槛使用

---

## 用户故事

### 普通用户
1. 打开应用看到「市场」Tab，按书体/书家浏览字帖
2. 搜索框输入"雁塔"立刻找到褚遂良雁塔圣教序
3. 点击订阅 → 字帖出现在仪表盘 → 开始学习
4. 退订后进度清除，重新订阅从零开始

### Admin（豪哥）
1. 用现有 APKG 导入功能创建牌组
2. 编辑牌组元数据（书家/朝代/书体/封面）
3. 点击「发布到市场」→ 所有用户可见
4. 在市场首页设置推荐字帖

## 需求池

### P0（必须）
| # | 需求 | 说明 |
|---|------|------|
| 1 | `marketplace_decks` 表 | 市场牌组元数据：name/calligrapher/dynasty/style/cover_image/featured/sort_order/description |
| 2 | `user_card_progress` 表 | 用户独立进度：user_id+card_id → interval/ease/repetitions/next_review/last_review |
| 3 | `user_subscriptions` 表 | 用户订阅关系：user_id+deck_id+subscribed_at |
| 4 | cards 表改造 | 去掉 SM-2 字段（interval/ease/repetitions/next_review/last_review），去掉 user_id |
| 5 | 市场页面 | `/market` 路由：书体分类 tabs + 书家筛选 + 搜索 + 牌组卡片网格 |
| 6 | 订阅/退订 API | POST /api/marketplace/:deckId/subscribe, DELETE /api/marketplace/:deckId/subscribe |
| 7 | 学习流程适配 | due-cards/new-cards 改为 JOIN user_card_progress，缺进度视为新卡 |
| 8 | 评分流程适配 | rateCard 写入 user_card_progress 而非 cards 表 |
| 9 | 导航入口 | PC 侧栏 + 移动底栏新增「市场」Tab |
| 10 | Admin 发布 | 牌组管理页新增「发布到市场」按钮 + 元数据编辑弹窗 |
| 11 | 权限控制 | 普通用户隐藏 APKG 导入导出、牌组创建；Admin 可见全部 |
| 12 | 数据迁移 | 现有 admin 牌组转为市场字帖，进度清零 |

### 导航变化

**PC 侧栏**：仪表盘 / **市场** / 牌组 / 统计 / 设置
**移动底栏**：Zi2Anki / **市场** / 牌组 / 数据 / 设置

### Admin 发布弹窗

牌组管理页 → 每个牌组行新增「发布到市场」按钮 → 弹窗：
- 书家（TextField）
- 朝代（TextField）
- 书体（Select: 楷/行/草/隶/篆）
- 封面图（从卡片中选一张 / 上传）
- 简介（TextField multiline）
- 推荐开关（Switch）
- [发布] / [取消]
